# Структура проекту

> Це **єдиний** `structure.md` у проекті. У вкладених папках `structure.md` не створюється. Тут живе повна мапа проекту.

---

## Ролі службових файлів у папках

У кожній тематичній папці (`project/marketing`, `brainstorm/marketing`, `project/developing` тощо) можуть лежати три типи службових файлів:

### `info.md`
Просто інформація **про саму папку**: що це за папка, що в ній зберігається, як вона працює, для чого існує. Відповідає на питання "що я тут знайду і як з цим працювати".

### `skill-main.md`
Опис **скіла / ролі**, яку AI повинен використовувати при роботі з цією папкою. Приклад: "ти копірайтер, пиши все з великої букви" → тоді AI у цій папці пише з великої букви. Або "ти React-розробник, всі пропси з великої букви" → AI у цій папці пише пропси з великої букви.

**Skill діє тільки під час роботи з файлами цієї папки та її вкладених папок.**

### `structure.md`
**Тільки один на весь проект — у `config/`.** У вкладених папках не створюється. Вся структура проекту описана тут, в одному місці.

---

## Правило успадкування (КРИТИЧНО)

**`info.md` і `skill-main.md` автоматично успадковуються всіма вкладеними папками.**

- Якщо `brainstorm/marketing/info.md` описує правила — вони діють і для `brainstorm/marketing/PAY/`, і для `brainstorm/marketing/YBT/`, і для будь-якої іншої вкладеної папки.
- Якщо `brainstorm/marketing/skill-main.md` задає роль — ця роль активна в усіх вкладених папках.
- Якщо вкладена папка має **власний** `info.md` або `skill-main.md` — вони **доповнюють** батьківські, а не замінюють. Конкретніший (вкладений) скіл має вищий пріоритет при конфлікті.

**Як це читається AI:**
Коли AI працює у `brainstorm/marketing/PAY/`, він читає ланцюжок:
1. `brainstorm/marketing/info.md` + `brainstorm/marketing/skill-main.md` (від батька)
2. `brainstorm/marketing/PAY/info.md` + `brainstorm/marketing/PAY/skill-main.md` (якщо існують)

## Повна мапа проекту

```
brainstorm-itlead/
├── AGENTS.md                     # точка входу для AI — читається завжди першим
├── CLAUDE.md                     # посилається на AGENTS.md
│
├── config/                       # правила роботи AI, джерело істини
│   ├── structure.md              # ← цей файл, єдина мапа проекту
│   ├── workflow.md               # алгоритм пам'яті, правила читання/запису
│   └── about-product.md          # опис продукту ITLead: фічі, тарифи
│
├── project/                      # РЕАЛЬНІ дані про проект (що вже є)
│   ├── info.md                   # загальний стан проекту
│   ├── timeline.md               # хронологія: зроблено / в роботі / заплановано / відхилено
│   ├── marketing/                # реальні маркетингові дані
│   │   ├── info.md
│   │   └── skill-main.md
│   └── developing/               # реальні дані про розробку
│       ├── info.md
│       └── skill-main.md
│
└── brainstorm/                   # ГІПОТЕЗИ, ІДЕЇ (ще не затверджено)
    └── marketing/                # маркетингові гіпотези
        ├── info.md
        ├── skill-main.md
        ├── PAY/                  # підтема: монетизація / платежі
        │   └── info.md
        └── YBT/                  # підтема: YouTube
            ├── info.md
            └── skill-main.md
```

---

## Правило розростання (масштабування пам'яті)

Коли в одній папці накопичується **7+ файлів** — AI створює `INDEX.md` у цій папці: один рядок на файл (назва + TL;DR у 1 речення). Це дозволяє читати тільки релевантні файли. До 7 файлів `INDEX.md` не потрібен.

## Куди додавати нове

- Нова тематична папка (наприклад `project/stats/`) — створити папку + `info.md` (опис зони) + опційно `skill-main.md` (якщо потрібна специфічна роль). **У config/structure.md (цей файл) оновити мапу.**
- Нова підтема всередині зони (наприклад `brainstorm/marketing/SEO/`) — створити папку + `info.md`. `skill-main.md` — тільки якщо потрібна роль, специфічніша за батьківську. Оновити мапу.
- Нова ідея / гіпотеза всередині підтеми — звичайний файл `.md` у відповідній папці. Мапу оновлювати **не треба** для окремих ідей.
